# 更新文件 Skill 指南

當使用者調用此 skill（「更新文件」）時，請遵循以下步驟來協助使用者更新本地端的專案檔案：

1. **執行 Git Pull**
   直接使用 `run_command` 工具在終端機執行以下指令來更新程式碼：
   ```bash
   git pull
   ```

2. **回報結果**
   終端機指令執行完畢後，將執行的結果回報給使用者：
   - 如果成功，告知使用者是否拉取了最新的變更（或者是已經處於最新狀態 `Already up to date.`）。
   - 如果遇到任何錯誤（例如 merge conflict、連線問題或未提交的變更導致無法 pull），請向使用者詳細說明錯誤原因，並主動詢問是否需要協助排除問題。
